Funktionen :

Funktion Mandant:
Es werden 2 Proceduren aufgerufen:
der eingegebene Mandant wird für die Erfassungsmaske vorbelegt und dann constraint Ist der eingegebene Mandant = 5 wird eine erweiterte Erfassungsmaske aufgerufen = + Feld: Festbis_Datum
In der Maske/Grid wird ein Scrollfeld mitgeführt. OB = muss erfasst werden UK = mit Auswahlmaske per Click oder mit der F4-Taste. Bei der Eingabe von 2023 wird das UK mit Nullen aufgefüllt, so dass der Wert dann 002023 ist. beim verlassen des Feldes wird die nachfolgende Funktion aufgerufen:
Function onValidating Returns Boolean

In dieser Function wird geprüft, ob das eingebene Konto mit dem erfassten Objekt vorhanden ist.
False = Nicht vorhanden: Es wird eine Fehlernachricht generiert und das Feld kann nicht verlassen werden.
True = vorhanden: das Konto wird im oberen Teil angezeigt sofern es nicht bereits aufgelöst ist (dann wird ebenfalls eine Fehlermeldung generiert).


Die Eingabe wird mit dem nächsten Feld fortgesetzt. Die Felder BS und ZW werden gegen die von Euch erfassten Daten in den CodeTabellen geprüft, so dass hier kein anderer als ein gültiger Wert erfasst werden kann. Aufgrund des erfassten Buchungsschlüssels wird das Textfeld vorbelegt (Auszahlung, Einzahlung, Storno usw.)

Das Datum der Wertstellung wird auf Wunsch von Herrn Fülbert nur noch auf das/die Vorhandene/n RJ geprüft. Ursprünglich lautete die Vorgabe +- 45Tage.

Text1: kann zu der Vorbelegung noch editiert werden.

Ist im Kopf der Maske Mandant 5 erfasst erscheint noch das Feld Festbis_Datum. Dieses wird wenn nichts erfasst wird automatisch gefüllt = +5 Jahre wegen der Strafzinsen.

Endprüfung:
Mit Verlassen dieses Feldes bzw. bei Mandant 1-4 mit dem Feld vorher wird die Erfassungszeile noch einmal validiert, bevor eine automatische Speicherung in der Datei Buchung erfolgt. Die Datei Konto wird ebenfalls gespeichert mit dem neuen Wert im Feld Hilfs-Saldo. Ein Teil der nicht direkt die Eingabe in den Feldern betrifft, wird dann beim Speichern der Werte in den Data_Dictionarys durchgeführt.

Was kann mit einer erfassten Buchung vor dem Tagesabschluss getan werden:
Die Buchung kann direkt im Grid (nicht mit dem (DBDEXPLORE) gelöscht werden. Bis auf OB und UK können alle Felder überschrieben werden. Das jeweilige Konto mit seinem Hilfs-Saldo wird entsprechend angepasst.
Journal kann direkt gedruckt /angesehen werden. Diese Hilfe kann angezeigt werden.
